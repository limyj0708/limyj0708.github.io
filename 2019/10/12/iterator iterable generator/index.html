Iterable, Iterator, Generator - 너희는 누구니?

Iterable

  • Iterator를 반환할 수 있다면, 어떤 오브젝트여도 Iterable함
    (Data structure일 필요는 없음)
  • 그럼 Iterator와 Iterable의 차이는 뭐지?

x는 iterable, y와 z는 iterator인 instance이며, iterable x로부터 값을 만들어낸다.
y와 z는 둘 다 상태를 저장하고 있다.
(next(y)를 두 번 호출할 시, 처음 한 번 호출한 상태를 기억하고 있다.)
x는 data structure지만(list), 꼭 data sturcture일 필요는 없다.

이는 class를 iterable인 동시에 자기 자신의 iterator로 만든다.
iterator로서 다른 오브젝트를 반환해도 전혀 상관 없다.

Iterator

그럼 도대체 Iterator가 뭐냐? next()를 호출하였을 때, 다음 값을 발생시키는 Stateful(상태를 기억하여 다음 동작에 활용이 가능한) helper object다.
(현재 몇 번째인지 기억해야 다음 값을 불러올 수 있을테니 Stateful이어야 함이 직관적으로 이해된다.)

__next__() 메소드를 가지고 있는 모든 오브젝트는 Iterator다.
어떻게 값을 발생시키는지는 상관이 없다.

Iterator는 값 공장이다. Iterator에 “다음 값”을 요청할 때마다,
Iterator는 내부 상태를 저장하고 있기 때문에, “다음 값”을 어떻게 계산해야 할 지 안다.

  2. 현재 호출을 위한 결과를 생성한다.

Central Idea : 게으른(lazy) 공장
밖에서 보면, iterator는 값을 요청하기 전까지 정지해 있는 게으른 공장처럼 보인다. 값 하나를 생산한 후에는
다시 정지 상태로 돌아가는.

Generator

드디어 목적지에 도차했다! Generator는 내가(글쓴이가) 완전 좋아하는 Python의 특징이다. Generator는 Iterator의 우아한 버전이다.

Generator는 Iterator를 상단의 피보나치 수열 Iterator처럼 작성할 수 있게 해준다. __iter()__와 __next()__를 가지는 클래스의 작성을 피하는, 우아하고 간결한 문법으로.

직관적으로 표현하면:

  1. 모든 Generator는 또한 Iterator이다. (반대는 아니다!)
  2. 모든 Generator는, 그러므로, 값을 게으르게 생산하는 공장이다.

뭐가 일어나는지 파헤쳐보자: 처음에는, fib가 평범한 Python 함수로 정의된다. 특별할 것이 없다.
그러나, 함수 안에 Return 키워드가 없다. 반환값은 Generator가 된다.

이제 f=fib() 가 호출되면, Generator가 instance화 되고 반환된다.
여기까지는 어떤 코드도 실행되지 않을 것이다. Generator는 초기의 정지 상태에서 시작한다.
더 직관적으로 보면, prev, curr = 0 은 아직 실행되지 않았다.

그리고, 이 Generator instance는 islice()에 감싸진다.
스스로 Iterator이기 때문에, 아직도 아무것도 일어나지 않는다.

그 다음엔, 이 Iterator는 이 Iterator의 모든 Argument들로부터 list를 생성할 list()에 의해 감싸진다. 그리고, 이 Iterator는 결과적으로 f instance에서 next()를 호출하는 islice() instance 위에서 next()를 호출하기 시작한다.

한 단계씩 보자. 최초의 실행에서, 이 코드는 결과적으로 prev, curr = 0,1 을 실행할 것이고, while True 루프에 들어가서, yield curr 구문을 마주한다. 이 구문은 현재 curr 변수에 있는 값을 생산하고
(현재 curr 변수에 할당된 값과 동일한 값을 반환한다는 의미) 다시 정지 상태가 될 것이다.

이 값은, 이 값을 생산할 islice() wrapper로 전달되고, list는 값 1을 리스트에 추가할 수 있게 된다.

그리고, list는 islice()에 다음 값을 요청한다. islice()는 f에 다음 값을 요청하고, f의 정지상태를 해제한다. f는 prev, curr = curr, prev + curr 구분 부터 다시 시작하게 된다. 이 다음에 while 루프의 다음 반복에 재진입하고, yield curr 구문을 만나고, 다음 curr 값을 반환한다.

이 동작은 Output list가 원소 10개를 가지고, list()가 islice()에 11번째 값을 요청하고, islice()가 끝에 도달했음을 의미하는 StopIteration 예외를 발생시킬때까지 일어난다. 그리로 list는 결과를 반환한다: 피보나치 수열 처음 10개 값을 가진 리스트를. Generator가 11번째 next() 호출을 받지 않았음에 주목해라. 결과적으로, 이 Generator는 다시 사용되지 않고, Garbage Collector에 의해 처리된다.

Generator function은 body에 yield를 가지고 있는 함수이다. 방금 이에 대한 예를 봤다.
