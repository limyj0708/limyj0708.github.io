  • 2606개 기업의 3년치 재무 데이터
    • 생존 기업은 2019/12/31 기준 최신 3년치, 상장폐지된 기업은 상장폐지일 기준 최신 3년치
    • 사용한 feature
      • 부채비율
      • 자기자본비율
      • 현금자산비율
      • 당기순이익율
      • 총자산회전율
      • 총자산증가율(전년동기)
      • 당기순이익증가율(전년동기)
  • 2606개 기업의 지난 3개 감사보고서 데이터
    • 사용한 feature

  • 상장시장 (코스닥, 코스피)

1-1. Oversampling

  • 전체 2606개 기업 중, 부도 기업은 전체의 16.04%
    • 불균형한 데이터(Imbalanced data)
    • Imbalanced data를 그대로 학습시키면, Recall이 낮게 나옴. Precision이 좀 낮아도 모든 부도 기업을 잡아내는 것이 좋은 부도 예측이라는 주제에서는, 허용할 수 없는 일
  • Imbalanced-learn 패키지를 사용하여, 소수 클래스를 증폭시키는 Oversampling을 진행
    • RandomOverSampling 사용 : 소수 클래스의 데이터를 반복해서 넣는 방법
    • 가상 데이터를 생성하는 Adaptive Synthetic Sampling, SMOTE 기법을 사용하지 못한 이유
      • 재무데이터에서 ‘수치를 계산할 수 없는 경우’에는, 구분을 위해 동종의 데이터가 절대로 절대로 가질 수 없는 큰 값을 할당하였음(100,000% 등)
        • 예 > [ 부채비율 = (유동부채 + 비유동부채)/자기자본 ] 에서,완전자본잠식이 되어 자기자본이 0이 된 경우
      • 가상의 값을 생성하게 되면, 등장해서는 안 되는 50,000%, 70,000% 등의 값이 생성됨

테크 기사를 비지도 학습으로 클러스터링 하기 (1) - K-means clustering

Dec 05, 2019 in 실전경험_기록

저번 글에서, Asyncio를 사용하여 약 40만개의 기사를 크롤링했다. (Verge 3만 5천개 + Cnet 36만개)
이 기사들을 클러스터링 하고 싶다. 어떤 경향성을 보일까?

이 주제에 대해 Austin L.E. Krause라는 사람이 수행한 기록과 코드가 있다. 이 사람이 분석을 어떻게 진행했는지 따라가면서, 원리를 이해하고 내가 가지고 있는 데이터에 적용해보자.

일단, K-Means Clustering 기법을 사용했다는데, 이것이 무엇인지부터 알아보자.

알고리즘 : K-Means Clustering

“the objective of K-means is simple: group similar data points together and discover underlying patterns. To achieve this objective, K-means looks for a fixed number (k) of clusters in a dataset.”

  • K-means 클러스터링의 목표 : 비슷한 데이터 포인트들을 모아서, 기저에 숨겨진 패턴들을 발견하는 것.
  • k : Centroid(중심)의 수.
  • Centroid(중심) : 클러스터의 중심을 나타내는 실제, 혹은 가상의 위치

  • 모든 데이터 포인트들은, 클러스터 내의 Sum of square가 줄어드는 방향으로 각 클러스터들에 할당된다.

    Sum of square =
    = The item in set
    = The mean of all items in set
    = The deviation of each item from the mean (편차 : deviation)

    이걸 좀 더 클러스터에 맞게 정의하면 이렇게 된다.

    • n개의 d-차원 데이터 오브젝트
    • k-mean 알고리즘은 이 n개의 데이터 오브젝트들을 k개의 집합 으로 분할한다. 를 집합 의 중심(centroid) 라고 하면,각 집합별 중심~집합 내 오브젝트간 거리의 제곱합(Sum of square)를 최소로 하는 집합 S를 찾는 것이 이 알고리즘의 목표이다. 이 개별 클러스터에 대한 이 값을 inertia라고 한다.

      Inertia is the sum of squared error for each cluster. Therefore the smaller the inertia the denser the cluster(closer together all the points are)

  • 는 와 벡터 사이의 euclidean distance를 의미한다.

K-Means 알고리즘은 어떻게 동작하는가

  • 중심들을 무작위로 생성한다. (사전에 정의된 k값에 따라)
  • 이 중심들은 클러스터들의 시작점으로서 사용된다.
  • 알고리즘은 중심의 값을 최적화하기 위해서 반복적으로 수행된다.
  • 클러스터의 생성, 최적화는 다음과 같은 경우에 멈춘다.
    • 중심이 안정화되었을 때. 즉, 클러스터링이 성공적으로 완료되어서 값에 더 변화가 없을 때.
    • 사전 정의된 반복 횟수만큼 진행되었을 때.

  • 위의 코드를 실행하면, 아래와 같은 결과값이 나온다.

    각각의 항목은 무슨 의미일까?

    • altorithm : 어떤 알고리즘을 사용했는가?

      algorithm “auto”, “full” or “elkan”, default=”auto”
      K-means algorithm to use. The classical EM-style algorithm is “full”. The “elkan” variation is more efficient by using the triangle inequality, but currently doesn’t support sparse data. “auto” chooses “elkan” for dense data and “full” for sparse data.

      • EM(Expectation-Maximization) style algorithm
        • 이 글을 본 다음에 (Likelihood)
        • 이 영상을 보자. (Maximum Likelihood)
        • 잠재 변수(latent variable)(직접 관측, 측정이 되지 않는 변수)를 가지고 있는 확률 모델에서, Maximum Likelihood Expectation을 구하는 기법 - 이라고 일단 알아두자. 참고
      • Elkan variation - 이를 최초로 제시한 논문.
      • sparse data, dense data - 설명이 잘 된 블로그

    • copy_x : 원본 데이터를 보존하느냐 마느냐. 기본값은 True(보존). False를 선택하면, 데이터의 C-contiguous를 보장할 수 없고, 이는 성능 하락을 일으킬 수 있다.

    • init : 초기화 방법. 기본값은 k-means++ 다.
      • k-means++ : 더 빠른 수렴을 위한 적절한 방법으로 초기 클러스터 중심을 선택한다는데, 자세한 내용은 이 논문에 있다. 언젠가 다 읽어볼 수 있었으면 좋겠다.
        • 간단한 설명은 여기 참고
      • random : 데이터에서 무작위로 선택함
      • ndarray : 이건 이 StackOverFlow 답변에 자세히 설명되어 있다.
      • 초기값 설정에 대한 좋은 글
    • max_iter : k-means 알고리즘을 한 번 실행(single run) 할 때 최대 반복횟수. 기본값은 300이다.
    • n_clusters : 생성하는 클러스터의 수
    • n_init : 각각 다른 중앙값 시드로 몇 번이나 알고리즘을 실행할 것인지 결정한다. 최종 결과는 실행한 값들 중 inertia가 가장 낮은 값이다.
    • n_jobs : 계산을 병렬로 돌릴 것인지, 병렬로 한다면 얼마나 프로세서를 사용할지에 대한 옵션. None이면 1로 보고, -1이면 모든 프로세서를 사용한다.
    • precompute_distances : 거리를 미리 계산함 - 더 빠르나 메모리를 더 많이 쓴다.
      • 기본값은 auto : n_samples * n_clusters > 1200만 이면 미리 계산하지 않는다. 이는 배정밀도(double precision) 사용 시, 1회당 100MB의 overhead와 동일하다.
      • True / False : 미리 계산을 항상 하느냐 마느냐.
    • random_state : 중앙값 초기 설정 시에 사용될 랜덤 넘버 시드를 설정한다.
    • tol : 허용 오차. 중심의 이동이 이 이하가 되면 알고리즘의 반복을 멈춘다.

      iteration stops when centroids move less than the tolerance.

    • verbose : 이 설명을 참고하면, 출력 메세지에 모델 디버깅에 도움이 될 만한 더 상세한 정보를 넣을지 말지에 대한 옵션이다.
