<기업부도예측-3> Decision Tree 기반 모델들 학습 및 평가(1)

Feb 05, 2020 in 프로젝트, 기업부도예측 (머신러닝/웹 리포트)

Table of Contents

  • 1. 학습에 사용한 데이터
    • 1-1. Oversampling
  • 2. 모델 학습
    • 2-1. StratifiedSuffleSplit
    • 2-2. 결과비교
      • 2-2-1. Oversampling 진행하지 않은 Raw 데이터
      • 2-2-2. RandomOversampling 진행한 데이터

제목에 (1)이 붙은 이유는, 수집한 모든 데이터를 사용하여 모델을 학습시킨 것이 아니기 때문.
Hyperparameter 튜닝도 진행하지 않았음.
(2)에서는 모든 데이터를 사용하여 학습하고, 결과가 가장 좋게 나온 모델에 Hyperparameter 튜닝도 진행할 예정.

1. 학습에 사용한 데이터

  • 상세 수집조건은 <기업부도예측-1> 프로젝트 개요, 수집 데이터 참조
  • 2606개 기업의 3년치 재무 데이터
    • 생존 기업은 2019/12/31 기준 최신 3년치, 상장폐지된 기업은 상장폐지일 기준 최신 3년치
    • 사용한 feature
      • 부채비율
      • 자기자본비율
      • 현금자산비율
      • 당기순이익율
      • 총자산회전율
      • 총자산증가율(전년동기)
      • 당기순이익증가율(전년동기)
  • 2606개 기업의 지난 3개 감사보고서 데이터
    • 사용한 feature
      • 지난 3개 감사보고서에서, ‘계속기업 불확실성’관련 키워드가 언급된 경우 1, 아니면 0으로 표시
  • 상장시장 (코스닥, 코스피)

1-1. Oversampling

  • 전체 2606개 기업 중, 부도 기업은 전체의 16.04%
    • 불균형한 데이터(Imbalanced data)
    • Imbalanced data를 그대로 학습시키면, Recall이 낮게 나옴. Precision이 좀 낮아도 모든 부도 기업을 잡아내는 것이 좋은 부도 예측이라는 주제에서는, 허용할 수 없는 일
  • Imbalanced-learn 패키지를 사용하여, 소수 클래스를 증폭시키는 Oversampling을 진행
    • RandomOverSampling 사용 : 소수 클래스의 데이터를 반복해서 넣는 방법
    • 가상 데이터를 생성하는 Adaptive Synthetic Sampling, SMOTE 기법을 사용하지 못한 이유
      • 재무데이터에서 ‘수치를 계산할 수 없는 경우’에는, 구분을 위해 동종의 데이터가 절대로 절대로 가질 수 없는 큰 값을 할당하였음(100,000% 등)
        • 예 > [ 부채비율 = (유동부채 + 비유동부채)/자기자본 ] 에서,완전자본잠식이 되어 자기자본이 0이 된 경우
      • 가상의 값을 생성하게 되면, 등장해서는 안 되는 50,000%, 70,000% 등의 값이 생성됨

2. 모델 학습

  • Decision Tree 기반의 다음 세 모델을 사용하였음
    • RandomForest
    • GradientBoosting
    • XGBoost
  • 각 모델들의 상세 설명은 다른 게시글에서 정리

2-1. StratifiedSuffleSplit

  • 정말 기가 막힌 운으로 데이터가 분할되어 성능이 좋게 나온 모델이 있었기 때문에, StratifiedSuffleSplit으로 Train set과 Test set 100건에 대한 검증 진행
  • 앞으로도 한참 써야 하기 때문에 별도의 모듈을 만들어서 진행
    • Oversampling, 학습, 예측, 평가결과 출력을 한 번에 진행함
    • 모듈 코드는 Github에서 확인 가능

2-2. 결과비교

  • 학습 코드는 Github에서 확인 가능
  • 요약 : 하나의 모델을 사용해야 한다면, 가장 Recall이 높은 GradientBoost-RandomOversampling 데이터 학습 모델을 선정

2-2-1. Oversampling 진행하지 않은 Raw 데이터

  • Raw 데이터에서는 RandomForest가 가장 Recall이 높음
  • F1-score는 세 모델이 거의 유사함
