Global Interpreter Lock (GIL)

Pythonм—җлҠ” Global Interpreter LockмқҙлқјлҠ” кІғмқҙ мһҲлӢӨ.

A global interpreter lock (GIL) is a mechanism used in computer-language interpreters to synchronize the execution of threads so that only one native thread can execute at a time.[1] An interpreter that uses GIL always allows exactly one thread to execute at a time, even if run on a multi-core processor. Some popular interpreters that have GIL are CPython and Ruby MRI.

Containers
Containers are an abstraction at the app layer that packages code and dependencies together. Multiple Containers can run on the same machine and share the OS kernel with other Containers, each running as isolated processes in user space. Containers take up less space than VMs (Container images are typically tens of MBs in size), can handle more applications and require fewer VMs and Operating systems.

VIRTUAL MACHINES
Virtual machines (VMs) are an abstraction of physical hardware turning one server into many servers. The hypervisor allows multiple VMs to run on a single machine. Each VM includes a full copy of an operating system, the application, necessary binaries and libraries - taking up tens of GBs. VMs can also be slow to boot.

A UNIX socket is an inter-process communication mechanism that allows bidirectional data exchange between processes running on the same machine.

IP sockets (especially TCP/IP sockets) are a mechanism allowing communication between processes over the network. In some cases, you can use TCP/IP sockets to talk with processes running on the same computer (by using the loopback interface).

UNIX domain sockets know that theyвҖҷre executing on the same system, so they can avoid some checks and operations (like routing); which makes them faster and lighter than IP sockets. So if you plan to communicate with processes on the same host, this is a better option than IP sockets.

Iterables vs Iterators vs Generators
лҘј ліҙкі  к·ёлҢҖлЎң мһ‘м„ұн•ҳмҳҖлӢӨ.

Iterable

  • IteratorлҘј л°ҳнҷҳн•  мҲҳ мһҲлӢӨл©ҙ, м–ҙл–Ө мҳӨлёҢм қнҠём—¬лҸ„ Iterableн•Ё
    (Data structureмқј н•„мҡ”лҠ” м—ҶмқҢ)
  • к·ёлҹј IteratorмҷҖ Iterableмқҳ м°ЁмқҙлҠ” лӯҗм§Җ?

xлҠ” iterable, yмҷҖ zлҠ” iteratorмқё instanceмқҙл©°, iterable xлЎңл¶Җн„° к°’мқ„ л§Ңл“Өм–ҙлӮёлӢӨ.
yмҷҖ zлҠ” л‘ҳ лӢӨ мғҒнғңлҘј м ҖмһҘн•ҳкі  мһҲлӢӨ.
(next(y)лҘј л‘җ лІҲ нҳём¶ңн•  мӢң, мІҳмқҢ н•ң лІҲ нҳём¶ңн•ң мғҒнғңлҘј кё°м–өн•ҳкі  мһҲлӢӨ.)
xлҠ” data structureм§Җл§Ң(list), кјӯ data sturctureмқј н•„мҡ”лҠ” м—ҶлӢӨ.

NOTE :
мӢӨмҡ©м Ғмқё мқҙмң лЎң, мў…мў… iterable classк°Җ iter()мҷҖ next()лҘј лӘЁл‘җ кө¬нҳ„н•ҳкі ,
selfлҘј л°ҳнҷҳн•ҳлҠ” iter()лҘј к°Җм§ҖлҠ” кІҪмҡ°к°Җ мһҲлӢӨ.
мқҙлҠ” classлҘј iterableмқё лҸҷмӢңм—җ мһҗкё° мһҗмӢ мқҳ iteratorлЎң л§Ңл“ лӢӨ.
iteratorлЎң self мқҙмҷёмқҳ лӢӨлҘё мҳӨлёҢм қнҠёлҘј л°ҳнҷҳн•ҙлҸ„ м „нҳҖ мғҒкҙҖ м—ҶлӢӨ.


к·ёлҹј мқҙм ң м•„лһҳмҷҖ к°ҷмқҙ м“°л©ҙ:

мӢӨм ңлЎңлҠ” мқҙлҹ° мқјмқҙ мқјм–ҙлӮңлӢӨ.

мқҙ мҪ”л“ңлҘј disassemble н•ҙ ліҙл©ҙ, iter(x)лҘј мӢӨн–үн•ҳлҠ” лҚ° н•„мҡ”н•ң GET_ITERмқҳ м§Ғм ‘ нҳём¶ңмқ„ ліј мҲҳ мһҲлӢӨ. FOR_ITERлҠ” next()лҘј л°ҳліөм ҒмңјлЎң нҳём¶ңн•ҳлҠ” кІғкіј к°ҷмқҖ лҸҷмһ‘мқ„ н•ҳлҠ” лӘ…л №м–ҙ(Instruction)лӢӨ. к·ёлҹ°лҚ° мқҙлҠ” мқҙм§„ мҪ”л“ңмқҳ нҳ•нғңлҘј ліҙм—¬мЈјм§ҖлҠ” м•ҠлҠ”лҚ°, interpreterмқҳ мҶҚлҸ„лҘј мң„н•ҙ мөңм Ғнҷ”лҗҳм–ҙ мһҲкё° л•Ңл¬ёмқҙлӢӨ.

Iterator

к·ёлҹј лҸ„лҢҖмІҙ Iteratorк°Җ лӯҗлғҗ? next()лҘј нҳём¶ңн•ҳмҳҖмқ„ л•Ң, лӢӨмқҢ к°’мқ„ л°ңмғқмӢңнӮӨлҠ” Stateful(мғҒнғңлҘј кё°м–өн•ҳм—¬ лӢӨмқҢ лҸҷмһ‘м—җ нҷңмҡ©мқҙ к°ҖлҠҘн•ң) helper objectлӢӨ.
(нҳ„мһ¬ лӘҮ лІҲм§ёмқём§Җ кё°м–өн•ҙм•ј лӢӨмқҢ к°’мқ„ л¶Ҳлҹ¬мҳ¬ мҲҳ мһҲмқ„н…ҢлӢҲ Statefulмқҙм–ҙм•ј н•Ёмқҙ м§ҒкҙҖм ҒмңјлЎң мқҙн•ҙлҗңлӢӨ.)

__next__() л©”мҶҢл“ңлҘј к°Җм§Җкі  мһҲлҠ” лӘЁл“  мҳӨлёҢм қнҠёлҠ” IteratorлӢӨ.
м–ҙл–»кІҢ к°’мқ„ л°ңмғқмӢңнӮӨлҠ”м§ҖлҠ” мғҒкҙҖмқҙ м—ҶлӢӨ.

IteratorлҠ” к°’ кіөмһҘмқҙлӢӨ. Iteratorм—җ вҖңлӢӨмқҢ к°’вҖқмқ„ мҡ”мІӯн•  л•Ңл§ҲлӢӨ,
IteratorлҠ” лӮҙл¶Җ мғҒнғңлҘј м ҖмһҘн•ҳкі  мһҲкё° л•Ңл¬ём—җ, вҖңлӢӨмқҢ к°’вҖқмқ„ м–ҙл–»кІҢ кі„мӮ°н•ҙм•ј н•  м§Җ м•ҲлӢӨ.

мҲҳл§ҺмқҖ Iteratorмқҳ мҳҲл“Өмқҙ мһҲлӢӨ. itertools н•ЁмҲҳл“ӨмқҖ IteratorлҘј л°ҳнҷҳн•ңлӢӨ.
лӘҮлӘҮ н•ЁмҲҳл“ӨмқҖ л¬ҙн•ңн•ң мҲңм„ң к°’л“Ө(sequences)мқ„ л°ңмғқмӢңнӮЁлӢӨ:


лӘҮлӘҮ н•ЁмҲҳл“ӨмқҖ мң н•ңн•ң мҲңм„ң к°’л“Өм—җм„ң л¬ҙн•ңн•ң мҲңм„ң к°’л“Өмқ„ л°ңмғқмӢңнӮЁлӢӨ:

  2. нҳ„мһ¬ нҳём¶ңмқ„ мң„н•ң кІ°кіјлҘј мғқм„ұн•ңлӢӨ.

л“ңл””м–ҙ лӘ©м Ғм§Җм—җ лҸ„м°Ён–ҲлӢӨ! GeneratorлҠ” лӮҙк°Җ(кёҖм“ҙмқҙк°Җ) мҷ„м „ мўӢм•„н•ҳлҠ” Pythonмқҳ нҠ№м§•мқҙлӢӨ. GeneratorлҠ” Iteratorмқҳ мҡ°м•„н•ң лІ„м „мқҙлӢӨ.

map(function, iterable, вҖҰ)

Return an iterator that applies function to every item of iterable, yielding the results. If additional iterable arguments are passed, function must take that many arguments and is applied to the items from all iterables in parallel. With multiple iterables, the iterator stops when the shortest iterable is exhausted. For cases where the function inputs are already arranged into argument tuples, see itertools.starmap.
